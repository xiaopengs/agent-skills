## 一、文章结构公式验证

本次使用 **「数据冲击→深挖→案例→升华」** 公式，效果验证如下：

### 段落节奏分配

```
数据冲击（前15%）  → 抓住注意力，建立"这件事值得看"的信号
深挖原因（中30%）  → "为什么是现在？"——从现象拽到本质
格局类比（中30%）  → npm 2015 类比 → 四层生态格局，展示视野
个人升华（后15%）  → "意味着什么？"→ 金句收尾
互动引导（末10%）  → 评论区提问 + 延伸阅读
```

### 🟢 做得好的

| 手法 | 文中位置 | 效果 |
|---|---|---|
| 数字冲击 | 开头「78,943颗星，单日涨3,392」 | 3秒抓人，不可绕过 |
| 自问自答 | 「为什么是现在？」「谁会赢？」 | 驱动阅读节奏 |
| 类比增厚 | npm 2015 → Agent Skills 2026 | 提供认知坐标系 |
| 四层格局 | 框架→库→基础设施→市场 | 展示"想清楚了" |
| 连续引用 | 引用自己前文「AI可以替我写…」 | 建立内容人格 |
| 金句收尾 | 「标准化Skill可以替你积累，但不能替你判断」 | 留记忆锚点 |
| 评论区提问 | 四个具体问题 | 引导互动 |

### 🟡 可改进的

| 问题 | 位置 | 改进方向 |
|---|---|---|
| 中段项目列举密度偏高 | 「不只pocock」段 | 可减少并列项目数，选最有代表性的2-3个 |
| 缺少作者个人经历 | 全文 | 如果能加入「我自己用skills的某次具体经历」会更真实 |
| 图片标注偏多 | 全文7处 | 可适当减少，改为文字描述 |

## 二、花叔风格落地检查清单

### 开头检查

- [x] 不要「今天我们来介绍…」→ 用「前几天在GitHub Trending上刷到一个数字」
- [x] 第一段 < 100 字
- [x] 有具体数字或具体场景
- [x] 展示情绪反应（「把我惊了一下」）

### 叙事节奏检查

- [x] 每 2-3 段有一个自问自答（「为什么是现在？」）
- [x] 展示思考过程而非直接给结论
- [x] 技术深挖不超过 2 段就拉回主线
- [x] 段长 2-4 句，不超过 5 句

### 细节密度检查

- [x] 具体数字：78,943 / 3,392 / 50K / 300K
- [x] 具体人名：mattpocock / Boris Cherny / Swyx / Guillermo Rauch
- [x] 具体项目：React / Vue / npm / ClawHub
- [x] 具体时间：2015 / 2014 / 2016

### 结尾检查

- [x] 有个人判断（「我的判断：最先成为默认选择的那个人赢」）
- [x] 有金句或变体（「标准化skill可以替你积累，但不能替你判断」）
- [x] 有互动引导（四个具体问题）
- [x] 有延伸阅读链接（裸URL，非Markdown格式）

## 三、微信 HTML 排版要点速查

### 必须遵守

```
✅ 所有样式 inline（style="..."）
✅ 只用 section / span / strong / em / br / img / a
✅ 正文 #333, 主色 #1a73e8, 强调 #ff6b35
✅ 图片占位 [图片] 用灰色文字标注
✅ 分割线只用一条（<section height:1px background:#eee>）
```

### 必须避免

```
❌ <style> 标签 / class 选择器
❌ flex / grid / shadow / transform
❌ 深色背景
❌ --- 分割线超过一条
❌ Markdown 链接格式 → 用裸 URL
```

### 小标题模板

```html
<section style="margin-top:30px;margin-bottom:16px;padding-left:12px;border-left:3px solid #1a73e8;">
  <span style="font-size:18px;font-weight:bold;color:#333333;line-height:1.6;">小标题</span>
</section>
```

### 金句引用模板


## 五、给后续创作的笔记

### 爆款公式选型指南

| 文章类型 | 推荐公式 | 示例选题 |
|---|---|---|
| 技术趋势分析 | 数据冲击-深挖-案例-升华 | Agent Skills、AI安全 |
| 个人经验分享 | 故事-冲突-转折-洞察 | 工作流搭建、工具实战 |
| 观点/态度 | 颠覆认知-新视角-重建 | "AI替你写但不能替你想" |
| 教程/指南 | 问题-原因-方案-升华 | Claude Code技巧、自动化 |
| 横向对比 | 对比-反差-原因-启示 | 多工具评测、多框架对比 |

### 最常见的3个翻车点

1. **开头用「今天我们来介绍」** → 改成具体场景/数字/感受
2. **技术深挖刹不住车** → 每2段技术就拉回"这意味着什么"
3. **结尾只总结不复盘** → 必须有个人判断 + 互动引导
