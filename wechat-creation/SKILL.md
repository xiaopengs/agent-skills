# 公众号内容创作指南

## 任务目标

本 Skill 用于：为 Jerry 的公众号内容提供全流程创作辅助

能力包含：
- 风格规范指导（10条必须做到 + 10条必须避免）
- 信息搜索增强（Follow-Builders AI Builder动态 + GitHub Trending + 行业新闻 + 网络搜索）
- 选题池管理（pool.json状态追踪 + 每日筛选 + 去重 + 过期机制）
- 选题讨论（提供3-5个选题方向及大纲 + 无偏见全貌评审）
- 内容创作（初稿撰写 + 三遍审校）
- 爆款结构优化（5种爆文公式 + 完读率关键）
- 文件组织管理（brief、知识库、协作文档、选题池、草稿）

触发条件：
- 需要创作公众号内容
- 需要讨论选题方向
- 需要审校优化已有内容
- 查看/管理选题池（选题池状态、pool.json）
- 公众号每日选题定时任务触发

## 前置准备

- 风格规范：阅读 [references/style-guide.md](references/style-guide.md) 了解 Jerry 公众号风格要求
- 模板参考：阅读 [references/templates.md](references/templates.md) 获取各类模板（选题讨论、协作文档、长文转X）
- 文件结构：按需创建目录（`_briefs/`、`_knowledge_base/`、`_协作文档/`、`草稿/`、`images/`）

## 操作步骤

### Step 1: 保存需求Brief

- 保存创作需求到 `_briefs/[项目名]-需求brief.md`
- 明确以下信息：
  - 目标受众
  - 核心信息
  - 字数要求
  - 特殊要求（如有）

### Step 2: 信息搜索（⭐必须优先执行）

**强制规则**：当用户提出新的产品/技术/概念时，必须先完成信息搜索，然后才能进行选题讨论。跳过此步骤直接选题是错误的流程。

**四大信息源（按优先级）：**

| 源 | 说明 | 用途 |
|---|---|---|
| Follow-Builders | 25位顶级AI Builder X动态 + 5个AI播客 | AI行业趋势、新产品、技术热点 |
| GitHub Trending | 每日热门开源项目 | 开发者社区关注焦点 |
| 行业新闻 | The Verge AI / 技术媒体 | 行业事件、政策、大公司动态 |
| 网络搜索 | 搜索引擎 / 官方文档 | 具体概念/产品的深度调研 |

- 对于新产品/新技术，必须进行信息搜索
- 搜索内容包括：官方文档、产品介绍、技术特性、用户评价、竞品对比
- 保存搜索结果到 `选题池/YYYY-MM-DD-info-search.md`
- 信息源盲区提醒：当前主要为英文信息流，如需中文社区视角应额外引入（知乎热榜、即刻、少数派等）
- 搜索完成确认：向用户确认搜索结果后再进入选题讨论

### Step 3: 选题讨论（必做）

**前置条件**：必须已完成Step 2信息搜索，对新概念/产品有充分了解后才能开始选题讨论。

提供3-5个选题方向，每个包含：
- 标题（吸引人、口语化，符合花叔风格）
- 核心角度（一句话说清切入姿势）
- 工作量评估（⭐评级）
- 大纲（3-7个大标题）
- 优劣势分析（优点 + 风险，不偏袒）
- 信息源标注（基于哪个源发现的话题）

**存入选题池**：所有选题必须在讨论后存入 `选题池/pool.json`，状态初始为 `draft`。

### Step 3.5: 选题池管理系统

#### 选题池文件结构

```
wechat_helper/选题池/
├── pool.json                    # 所有选题主池（JSON）
├── INTEGRATION.md               # 信息源集成方案
├── YYYY-MM-DD-info-search.md    # 每日信息搜索结果
└── YYYY-MM-DD.md                # 每日选题记录
```

#### pool.json 数据结构

```json
{
  "id": "topic-YYYYMMDD-NNN",
  "title": "选题标题",
  "angle": "一句话核心角度",
  "outline": ["大纲项1", "大纲项2", ...],
  "source": "信息源标注",
  "difficulty": "⭐ | ⭐⭐ | ⭐⭐⭐",
  "status": "draft | selected | published | expired",
  "createdAt": "ISO时间戳",
  "selectedAt": "ISO时间戳或null",
  "publishedAt": "ISO时间戳或null"
}
```

#### 核心规则

| 规则 | 说明 |
|---|---|
| **去重** | 与已有选题标题相似度高的跳过，不重复创建 |
| **过期机制** | draft 状态超过 14 天的选题自动标记为 `expired` |
| **永不重选** | status=`published` 的选题永不再次选择 |
| **每日重置** | 每天重新从未发布池中筛选，不延续前一天的选择 |
| **信息新鲜度** | 搜索必须是最新数据，不依赖旧缓存 |

#### 每日选题筛选优先级

1. 信息源新鲜（当天拉取的最新趋势）
2. 角度独特（与已有内容不重复）
3. 难度适中（优先⭐⭐，其次⭐或⭐⭐⭐）
4. 受众匹配（面向 AI Native Coder）
5. 池龄短（避免选题在池中存放大久失去时效性）

#### 无偏见全貌评审要求

以下内容**必须**在选题后推送给用户，不得跳过：

**📊 池子总览**
- 总数 / 今日新增 / 已选 / 草稿 / 已发布 / 过期 各多少

**✅ 今日入选（2个）**
- 每个入选选题：标题、角度一句话、大纲要点、入选理由
- 入选理由基于：信息源、趋势吻合度、受众匹配度的客观判断

**📋 完整待选池（所有 draft 状态选题）**
- 每个 draft 选题：标题、角度、难度、创建时间、池龄天数
- 每个选题的客观评估：**优点 + 风险/不足**，不偏袒

**⚠️ 无偏见底线**
- 每个选题都给出优点和风险，**不刻意美化入选的**
- 如果某个 draft 选题其实比今天入选的更值得写，**直接指出**
- 如果今天的信息源不够丰富、选题质量整体偏弱，**也直接说明**
- 指出当前信息源的盲区（如缺乏中文社区视角）

**📌 审核说明**
- 审核不是形式——如果选题质量不达标，该说就说
- 用户有最终决定权：入选的可以被否，备选的可以被激活
- 选题池推送后立即停止，等待用户确认哪篇进入创作

### Step 5: 学习风格

- 阅读 `../写作参考/` 文件夹中的参考文档
- **从 [assets/](assets/) 目录中随机选择2-3篇 Jerry 的历史文章进行阅读**
- 理解 Jerry 风格的关键特征

### Step 6: 使用个人素材库

- 加入真实经历降低AI味
- 引用具体的时间、地点、人物、事件
- 保持内容的真实感和可信度

### Step 7: 创作初稿

- 基于真实数据和搜索信息
- 保持「实践+落地」调性
- 初稿可不完美，后续审校环节优化

### Step 8: 三遍审校

- 第一遍：检查基本错误（错别字、标点、格式）
- 第二遍：优化表达和逻辑
- 第三遍：风格检查和内容升华

### Step 9: 爆款结构优化

**5种爆文公式**：
1. 问题-原因-方案-升华（教程类）
2. 故事-冲突-转折-洞察（经验类）
3. 颠覆认知-新视角-重建（观点类）
4. 对比-反差-原因-启示（对比类）
5. 数据冲击-深挖-案例-升华（分析类）

**完读率关键检查**：
- 开头（前100字）：3秒抓住、悬念设置、价值承诺
- 中段：每300字有新洞察、2-3个转折
- 结尾：升华主题、行动引导

### Step 10: 配图

- 推荐5-8张，最少3张
- 重要的概念、产品界面、数据图表等需要配图
- 在文中标注 [图片]

### Step 11: 长文转X（可选）

**标准结构**（200-500字）：
1. 开头（30-50字）：金句/数据/价值主张
2. 展开（100-300字）：具体细节、个人经历
3. 结尾（20-30字）：互动引导

**X长文版**（1000字+）：
- 用"——"分隔章节
- 重要人名第一次@账号
- 结尾开放式问题

## 注意事项

- 仅在需要时读取参考，保持上下文简洁
- **流程顺序强制**：遇到新概念/产品，必须先执行Step 2信息搜索，再执行Step 3选题讨论，严禁跳过搜索直接选题
- **选题池管理**：所有选题必须存入 pool.json，按状态追踪；已发布的永不重选；draft超过14天自动过期
- **每日筛选独立**：每天重新从未发布池中重新筛选，不沿用前一天的选择
- **无偏见评审**：选题讨论后必须推送全貌报告（含每个选题的优点和风险），不得只报入选的
- 不要直接开始写文章，必须先进行选题讨论
- 严格遵守风格规范，避免AI腔
- 每篇文章都要进行三遍审校
- 充分利用智能体的语言理解和创作能力，所有环节由智能体指导完成

## 自动化定时流程

通过 `openclaw cron` 创建每日定时任务：

```
名称：公众号每日选题
时间：每天 08:00 Asia/Shanghai
模式：isolated 独立会话
```

自动执行流程：
1. 拉取四大信息源最新数据
2. 识别趋势，保存搜索记录
3. 生成 3-5 个新选题，追加到选题池
4. 清理过期选题（>14天draft）
5. 从整个未发布池中筛选 2 个选题
6. 推送选题池全貌报告（含无偏见评审）
7. 等待用户确认 → 进入创作流程

## 使用示例

### 示例1：个人文章创作

**功能说明**：创作个人经验分享类文章

**执行方式**：
- 智能体指导选题 → 结合个人经历和行业观察
- 智能体创作 → 使用心路历程式叙事、自问自答式叙事
- 智能体优化 → 应用爆款结构公式

**关键要点**：
- 加入具体的时间、地点、人物
- 展示决策过程（犹豫→思考→决定）
- 数据后加个人判断（"这个数据还挺猛"）

### 示例2：审校优化

**功能说明**：对已有内容进行审校和优化

**执行方式**：
- 智能体三遍审校 → 基本错误、表达优化、风格检查
- 智能体应用爆款公式 → 选择合适的结构框架
- 智能体检查完读率 → 开头抓人、中段转折、结尾升华

**关键要点**：
- 开头前100字要有悬念或价值承诺
- 中段每300字要有新洞察
- 所有案例都要有链接，重要内容标注[图片]
