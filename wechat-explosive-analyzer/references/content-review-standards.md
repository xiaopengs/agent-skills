# 内容审核标准清单（2025-2026版）

## 一、平台推荐机制

### 三层分发漏斗

**第一层：合规审核**
- 原创度检测：相似度过高会被降权
- AIGC检测：AI味过重会被限流或标记为低质量
- 违规过滤：硬广软文、标题党、敏感词、分组发布直接失去推荐资格

**第二层：初始流量池测试**
- 文章通过后系统先推给约100-500人测试
- 核心指标及格线：
  - 点击率(CTR)：> 3%（标题+封面决定）
  - 完读率/完播率：> 70%（结构+节奏决定）
  - 互动率(点赞+评论+转发+收藏)：> 5%
  - 读后关注率：> 1%

**第三层：社交裂变**
- 「看一看」推荐：好友互动（尤其是「在看」）权重极高
- 朋友圈传播：标题必须适合朋友圈场景，不能让转发者产生社交压力
- 搜一搜流量：标题和正文关键词密度影响搜索排名

### 视频号与公众号差异

| 维度 | 公众号文章 | 视频号视频 |
|------|-----------|-----------|
| 核心钩子 | 标题 + 封面图 | 封面帧 + 前3秒 |
| 完成信号 | 滚动深度 / 阅读时长 | 完播率、重播次数 |
| 互动权重 | 评论 > 转发 > 点赞 | 转发 > 评论 > 点赞 > 收藏 |
| 裂变引擎 | 「在看」+ 朋友圈 | 转发到聊天 + 朋友圈 |
| 算法窗口 | 24-72小时 | 2-6小时（更短更快） |
| 社交可见性 | 阅读量（如开启） | 点赞/评论数始终可见 |

> 视频号关键指标：**转发÷点赞 > 2:1** 是爆款潜力的强烈信号。

### 账号标签机制

- 平台根据历史内容关键词、行业属性、互动画像给账号打标签
- 低粉账号突围关键：**选题足够垂直、关键词足够精准**
- 正文中自然埋入长尾关键词（如「AI编程工具链」「程序员效率」「代码质量」）有助于分类

### 2025年平台重点打击的「AI味」信号

- 段落长度过于均匀（每段都是3-4行）
- 过度使用「首先/其次/最后」「值得注意的是」「综上所述」
- 完美规整的「总-分-总」结构，章节编号一、二、三、四、五、六
- 缺乏具体个人故事，只有泛泛而谈的「我们」「很多人」
- 每个章节大量使用「•」符号列表
- 全文没有出现具体人名、地名、产品名、金额、日期

## 二、六维评分标准

### A. 标题审核（权重30%）

**必须检查项**：
- [ ] 包含**具体人群标签**（如「月薪3万的程序员」「35岁打工人」「刚毕业的运营」）
- [ ] 包含**痛点或好奇心缺口**（如「为什么越努力越穷」「面试挂了」）
- [ ] 包含**数字、时间点或名人**增强可信度
- [ ] 长度在13-28个汉字之间（手机端最优）
- [ ] 不含需要反应超过1秒的英文单词（除非目标读者明确为双语人群）
- [ ] 不含生僻文学典故或文化梗，避免自嗨式窄众表达
- [ ] 转发到朋友圈不会让老板/同事皱眉头

**2025年已验证的爆款标题公式**：
1. **人群 + 反常结果 + 悬念**：「那个每天写500行AI代码的同事，试用期没过」
2. **数字 + 痛点 + 解决方案**：「每天被会议吃掉4小时的程序员，我用这3个脚本偷回来了」
3. **热点产品/新闻 + 垂直角度**：「Cursor 0.45发布后，我决定不再手写CSS了」
4. **反常识 + 身份锚定**：「越会用AI补全的程序员，越容易被裁员（不是危言耸听）」
5. **名人背书 + 冲突 + 反转**：「马斯克转发的AI失业名单，DeepSeek说框架全错了」

---

### B. 开头钩子审核（权重20%）

**3秒法则**：前50字内必须出现以下**至少一种**：
- **shocking 数据 + 时间/场景锚定**：「2026年3月，Karpathy的一份表格让6000万白领失眠」
- **个人危机 + 具体细节**：「我用Cursor写了8个月代码，上周面试一道简单算法题没写出来」
- **权威反常识论断**：「OpenAI联合创始人说：软件工程师是最危险的职业」
- **感官化场景 + 情绪张力**：「凌晨2点，我删掉了写了3天的AI生成代码」

**开头致死红线**（导致3秒内划走）：
- 用反问句开场：「你有没有想过……」（信息密度为0）
- 缓慢铺垫背景：「最近AI很火，很多人都在讨论……」
- 情绪空洞：「我很焦虑」（不说具体因为什么焦虑）
- 首段就开始下定义或讲历史

---

### C. 选题切口审核（权重25%）

**2025-2026年高热度技术/职场选题赛道**：
1. AI工具替代/提效具体岗位（Cursor、Copilot、Claude Code、Trae）
2. 35岁危机与裁员（真实故事+数据）
3. AI时代人类能力退化（可验证的个人实验）
4. 独立开发者/一人公司（具体收入+工具栈）
5. AI时代薪资/offer谈判（具体数字、公司、城市）
6. 名人AI言论的本地专家反驳（马斯克、Altman、Karpathy + 本土AI视角）

**选题深度测试**：
- ✅ 好案例：「Cursor的3个隐藏功能让我每周少加班10小时」（工具具体、收益具体）
- ❌ 差案例：「AI时代如何保持竞争力」（太宽，算法找不到入口）
- ✅ 好案例：「我把Karpathy的342职业表发给DeepSeek，它说框架全错了」（触发事件具体、冲突具体）
- ❌ 差案例：「AI对职业的影响分析」（这是论文题目，不是朋友圈传播品）

---

### D. 结构与节奏审核（权重15%）

**公众号文章结构要求（反AI味）**：
- 段落长短必须参差：1行的冲击力句子，与4-5行的解释性段落混合出现
- 过渡用语用「说实话」「说句扎心的」「你可能不信」「我当时气得差点摔键盘」，替代「首先/其次/最后」
- **每200-300字必须出现一次具体细节**：精确日期、产品版本号、金额数字、城市名、公司名、人名
- 打破「总-分-总」：用叙事悬念渐进释放答案，不要开篇就亮底牌
- 结尾必须是一个**向读者抛出的问题**，或一句**必须评论才能接话的反常识结论**

**视频号结构要求（前15秒定生死）**：
- 画面/台词第1-3帧必须输出一个与观众预期相反的钩子
- 禁止开场：「大家好我是XX」「欢迎来到我的频道」「最近有一个新闻」
- 必须直接放出整期视频中最具争议性的一句话
- 信息密度：每15-20秒必须出现一个新的信息点或情绪节拍

---

### E. 社交货币审核（权重10%）

**核心问题**：读者为什么要把这篇内容转发给一个**具体的人**或**具体的群**？

| 转发动机 | 内容特征 |
|---------|---------|
| 「我就说吧」 | 验证了转发者之前的观点 |
| 「开会前赶紧看」 | 对同事有行动上的紧迫感 |
| 「原来我们不是一个人」 | 在群里提供情绪共振 |
| 「这个数据太吓人了」 | 让转发者显得是信息源 |
| 「太真实了，笑着笑着哭了」 | 用幽默或共情建立人际连接 |

**社交货币致死行为**：
- 结尾直接写「请转发给你的朋友」——转化率暴跌30%以上
- 内容让读者看起来懒惰、无知、只会抱怨
- 标题已经把结局说完了（没有悬念就没有点击）

---

### F. 分内容类型专项清单

**程序员/技术类内容**
- [ ] 包含具体工具名称（Cursor、Copilot、Windsurf、Trae、Claude Code）
- [ ] 包含具体版本号或日期（如适用）
- [ ] 包含可量化的结果（节省了多少时间、代码行数变化、错误率变化）
- [ ] 直面一个恐惧（被替代、技能退化、跟不上技术迭代）

**职场/焦虑类内容**
- [ ] 锚定7天内发生的最新事件或数据发布（时效性越强越好）
- [ ] 包含「我们 vs 他们」的对抗框架（员工 vs 管理层、个人 vs 系统）
- [ ] 提供的解决方案不是鸡汤（不是"只要努力就能赢"）
- [ ] 结尾给的是情绪出口或一个极小的具体动作

**辟谣/反常识/打脸类内容**
- [ ] 清楚陈述被反驳的原始观点及其来源
- [ ] 用一个出人意料的权威来挑战它（不是作者本人跳出来反驳）
- [ ] 叙事结构：原始观点 → 为什么它看起来对 → 隐藏的漏洞 → 更深层的真相
- [ ] 「纠正后的结论」必须比「原始观点」更有趣、更反常识
